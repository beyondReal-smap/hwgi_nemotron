1. PersonaFit 개요

PersonaFit은 보험·금융 상품기획자(FP)와 마케터를 위해 "이 상품에 누가 반응할 것인가"를 정량 데이터로 답해주는 의사결정 보조 도구입니다. 상품설명서를 텍스트·파일로 입력하면, LLM이 소구점·타겟 5차원을 추출하고 100만 합성 페르소나와 임베딩 유사도로 매칭해 반응도·지역 분포·인사이트 리포트를 분 단위로 산출합니다.

왜 합성 페르소나인가?

실제 가입자 데이터는 PII·내부 규정으로 외부 분석에 제한적입니다. NVIDIA Nemotron-Personas-Korea는 KOSIS·대법원·국민건강보험공단·KREI 통계를 기반으로 인구통계가 정합한 합성 데이터이며 CC BY 4.0로 상업적 활용이 가능합니다. 실제 인물·가입자와는 무관하므로 PoC·기획 단계에서 빠르게 시나리오를 검증할 수 있습니다.

4가지 핵심 기능

  • 상품 분석 — 약관·설명서 → 소구점 추출 → 타겟 매칭 → 마크다운 리포트 자동 생성
  • 현황 대시보드 — 100만 페르소나의 인구통계·지역·직업·텍스트 길이 분포 시각화 + 자연어 검색
  • 페르소나 탐색 — 메타 필터 + 자연어 조건을 조합해 타겟 그룹을 만들고 세그먼트로 저장
  • 설문 시뮬레이션 — 저장된 세그먼트의 페르소나에게 LLM이 가상 설문 응답을 생성, 차트 리포트 산출

5. 핵심 프로세스 4종

5-A. 상품 분석 POST /api/analyze

1
입력 추출
텍스트 또는 파일(PDF·HWP·DOCX) 업로드 → 본문 텍스트 추출
2
소구점 추출
Claude Sonnet이 tool_use로 SellingPoints(연령/성별/가구/학력/직업 + 키워드 + 카테고리 가중치) JSON 산출
3
룰 사전 필터
store에서 메타 조건 일치 후보 추출. 200 미만이면 자동 완화
4
임베딩 정렬
소구점 텍스트 → 1536d 임베딩 → 후보 대상 dot product → 상위 K
5
점수 결합
0.7×cos + 0.2×rule + 0.1×category → 0~100 정규화
6
집계 + 리포트
시도/시군구 분포 · cohort 3종 → Claude Haiku가 FP용 마크다운 리포트 생성

5-B. 자연어 페르소나 검색 POST /api/dataset/personas/search · /filter

"30대 워킹맘", "은퇴 후 등산을 좋아하는 60대 남성" 같은 자연어로 페르소나 후보를 찾는 하이브리드 파이프라인. 순수 코사인은 짧은 인구통계 쿼리에서 점수가 낮게 나오는 한계가 있어, LLM 메타 추출 + 후보 축소 + 잔여 의미 임베딩의 3단 흐름으로 해결합니다.

1
메타 추출
Claude Haiku/Qwen이 tool_use로 sex/age/region/has_children/occupation/employment 분해
2
후보 필터
store.filter_indices로 메타 일치 후보군 추출 — '검색에 부합하는 사람' 수
3
잔여 의미 임베딩
메타로 흡수 안 된 잔여 텍스트("등산", "야간 근무" 등)만 임베딩하여 유사도 정렬
4
컷 + Fallback
잔여 의미가 있으면 ≥0.3 컷. 0건이면 메타 결과로 fallback (사용자에게 0명 안 보임)

실제 결과 ("30대 워킹맘")

이전: 순수 코사인 → 0/0/2명 (≥0.5/0.4/0.3) — 매칭 실패.
이후: 하이브리드 → 메타 추출 {sex:여자, age:30-39, has_children:true, employment:employed} → 25,526명 매칭. 잔여 텍스트가 빔(워킹맘은 메타로 완전 흡수) → 결과는 의미 유사도 내림차순.

5-C. 세그먼트 빌더 /api/segments

1
필터 조합
/personas 페이지에서 메타(연령·성별·지역·직업·가구) + 자연어 조건
2
개별 선택
결과 카드에서 페르소나 체크박스로 정밀 선택 (최대 10,000명)
3
세그먼트 저장
이름·설명·필터·persona_uuids를 묶어 세그먼트로 영속화
4
설문에 재사용
설문 마법사에서 세그먼트 불러오기 → 동일 그룹에 반복 조사

5-D. 설문 시뮬레이션 /api/surveys/{id}/run

1
설문 작성
단일/다중 선택·척도·NPS·주관식 5종 질문 유형. Excel/Word/CSV 일괄 업로드, AI 추천도 지원
2
타겟 결정
세그먼트 불러오기 또는 즉석 필터로 응답 대상 페르소나 풀 확정
3
병렬 시뮬레이션
survey_engine이 (persona × question)마다 LLM 호출. 캐시·재시도·진행률 추적
4
차트 리포트
질문별 분포·confidence + Claude Haiku가 총평·인사이트 생성 → CSV 내보내기 지원

6. 활용 방법 (페이지별)

① 분석 페이지 /

  • 상품설명서·약관을 텍스트 박스에 붙여넣거나 PDF/HWP 파일을 드래그앤드롭
  • "분석 시작" → 20초 내외에 ScoreCard·PersonaList·RegionChart·ReportPanel 4개 패널이 채워짐
  • 리포트는 마크다운으로 복사 가능 · 분석 결과는 자동 이력 저장
  • LLM 토글로 사내 sLLM(무료) ↔ Claude(고품질) 즉시 전환

② 현황 페이지 /overview

  • 9개 섹션 — 핵심 지표 → 지역 지도 → 인구통계 7종 → 직업군 → 페르소나 텍스트 → 자연어 검색 → 출처
  • 분모 토글: 직업군 분포를 "전체" vs "취업자만"으로 즉시 전환 (무직 36.7% 왜곡 방지)
  • 자연어 검색 — "30대 워킹맘"처럼 입력하면 메타 추출 칩 + 매칭 인원 + 상위 페르소나 노출

③ 페르소나 탐색 /personas

  • 좌측 사이드바: 연령·성별·지역·가구·학력·직업 필터 + 자연어 조건
  • 우측: 매칭 요약(성별/연령대/시도 분포) + 카드/테이블 토글로 페르소나 24명씩 페이지
  • 체크박스로 페르소나 개별 선택 → 하단 sticky CTA "세그먼트로 저장"
  • 카드 클릭 시 풀 프로필 모달 (7개 카테고리 페르소나·skills·hobbies·career)

④ 설문 /surveys

  • 설문 마법사: 제목·목적 → 타겟(세그먼트 또는 즉석 필터) → 질문(직접/엑셀/AI 추천)
  • "시뮬레이션 시작" → 진행률 페이지에서 실시간 응답 생성 추적
  • 리포트 페이지: 질문 유형별 시각화(파이/막대/히스토그램) + Haiku 자동 총평 + CSV 다운로드

활용 시나리오 예시

  1. 신상품 콘셉트 검증 — "이 약관에 정말 30대 워킹맘이 반응할까?" → /analyze로 타겟 5차원·반응도 산출
  2. 채널 마케팅 지역 우선순위 — RegionChart의 시군구 상위 5개를 광고·영업 자원 배분 근거로
  3. 경쟁 약관 비교 — 같은 카테고리 두 상품을 분석해 카테고리 가중치·타겟 폭 차이를 정량 비교
  4. 가상 인터뷰 — 세그먼트에 NPS·만족도 설문을 돌려 카피·약관 문구 A/B 후보 결정
