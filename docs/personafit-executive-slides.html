PersonaFit
Business Problem

왜 필요한가

보험 상품은 복잡하고, 타겟 판단은 늦고, 영업 메시지는 현장 경험에 의존합니다.

상품기획

약관과 설명서에서 가입 조건, 보장 포인트, 타겟 고객을 사람이 읽고 정리합니다. 회의마다 가설이 달라집니다.

마케팅

어느 연령·지역·직군에 먼저 예산을 쓸지 빠르게 비교하기 어렵습니다. 캠페인 전 실험 비용이 큽니다.

FP 영업

고객 유형별 첫 문장, 반론 대응, 관심 포인트가 표준화되지 않습니다. 숙련자 경험이 팀 전체로 확산되지 않습니다.

이 프로젝트는 실제 계약 가능성을 단정하지 않습니다. 대신 “어떤 고객군을 먼저 검토해야 하는가”를 데이터 기반으로 좁히는 역할을 합니다.

Nemotron-Personas-Korea

데이터셋 설명

Nemotron-Personas-Korea는 한국 사회의 분포를 반영하도록 만든 합성 한국인 페르소나 데이터입니다.

실제 고객 명단이 아니라, 공공 통계와 도메인 지식을 바탕으로 생성한 가상 인물 집합입니다. 따라서 개인정보 없이도 “한국 고객처럼 다양한 반응”을 실험할 수 있습니다.

규모

100만 레코드 안에 7개 페르소나 서술이 들어 있어 총 700만 개의 페르소나 텍스트를 제공합니다.

범위

17개 시도와 252개 시군구, 성별·연령·교육·직업·가구 등 한국형 인구통계 축을 포함합니다.

출처 기반

KOSIS, 대법원, 국민건강보험공단, 한국농촌경제연구원, NAVER Cloud의 자료·전문성이 반영되었습니다.

라이선스

CC BY 4.0 기반으로 공개되어 출처 표기 조건에서 상업적 활용과 내부 PoC 검토가 가능합니다.

구성 예시 업무 활용 의미

7개 페르소나 텍스트 요약, 직업, 스포츠, 예술, 여행, 음식, 가족 페르소나 상품 혜택과 생활 관심사가 얼마나 잘 맞는지 의미 기반으로 찾습니다.
부가 속성 문화적 배경, 기술·전문성, 경력 목표, 취미·관심사 “왜 반응하는지”를 설명하는 근거 문장과 영업 화법을 만듭니다.
개인정보 성격 실제 인물이 아닌 완전 합성 데이터 사내 테스트·기획 검토 단계에서 개인정보 리스크를 낮춥니다.
중요한 차이는 “통계표”가 아니라 “사람처럼 읽히는 서술형 프로필”이라는 점입니다. 그래서 숫자 분포와 고객 맥락을 동시에 다룰 수 있습니다.

Applied Technology

기술을 임원 관점으로 번역하면

문서 이해, 대규모 유사 고객 검색, 결과 설명을 역할별 기술로 나눴습니다.

기술 프로젝트에서 하는 일 업무 가치
대형언어모델 약관에서 혜택, 가입 조건, 타겟 키워드를 구조화합니다. 사람이 긴 문서를 읽고 표로 정리하는 시간을 줄입니다.
임베딩 상품 설명과 페르소나 서술을 숫자 벡터로 바꿔 의미가 가까운 고객을 찾습니다. 단순 키워드가 아니라 생활 맥락까지 반영한 매칭이 가능합니다.
룰 기반 점수 연령·성별·가구·학력·직업처럼 명시 조건을 점수에 반영합니다. 약관상 가입 조건을 놓치지 않고 결과를 설명할 수 있습니다.
인메모리 분석 100만 행을 메모리에 올려 빠르게 전체 점수화합니다. 회의 중에도 여러 상품 가설을 반복 비교할 수 있습니다.
웹 대시보드 분석, 이력, 지도, 페르소나 탐색, 설문 시뮬레이션을 화면으로 제공합니다. 개발자가 아닌 실무자가 직접 결과를 확인합니다.

Business Process

업무 흐름

실무자는 상품 문서만 넣고, 시스템은 타겟 전략 초안을 6단계로 만듭니다.

1 상품 문서 입력 약관·상품설명서·제안서를 붙여넣거나 파일로 업로드합니다.
2 타겟 조건 추출 나이, 성별, 가구, 학력, 직업, 핵심 혜택을 표준 구조로 바꿉니다.
3 100만 명 점수화 상품과 비슷한 생활 맥락을 가진 페르소나를 전체에서 찾습니다.
4 지역·계층 집계 시도, 시군구, 연령, 직업, 가족 형태별 반응 분포를 계산합니다.
5 의견 생성 상위·하위 페르소나가 상품을 어떻게 받아들일지 짧은 의견을 만듭니다.
6 전략 리포트 공략 타겟, 우려 요인, 영업 화법, 지역 우선순위를 정리합니다.
기획 회의 전 타겟 가설 초안을 자동 생성하는 흐름입니다. 08

Scoring Logic

점수는 어떻게 믿을 수 있나

반응도는 의미 유사도, 가입 조건, 관심 카테고리를 합쳐 계산합니다.

70%

의미 유사도

상품의 혜택·키워드와 페르소나의 생활 서술이 얼마나 가까운지 계산합니다.

20%

명시 조건

연령, 성별, 가구, 학력, 직업이 약관 조건과 맞는지 보너스를 줍니다.

10%

관심 영역

가족, 직업, 여행, 음식 등 상품과 관련 깊은 페르소나 카테고리를 반영합니다.

Risk & Governance

임원 관점의 통제 포인트

실무 적용 전에는 “합성 데이터의 한계”와 “실제 성과 검증”을 분리해 관리해야 합니다.

합성 데이터 한계

실제 가입자 데이터가 아니므로 결과는 시장 가설입니다. 실제 통계와 교차 검증해야 합니다.

전환율 한계

반응 가능성이 실제 가입을 의미하지 않습니다. 캠페인 A/B 테스트가 필요합니다.

설명 책임

점수만 제시하면 위험합니다. 페르소나 근거, 조건, 지역 분포를 함께 보여줘야 합니다.

운영 보안

운영 전 인증, 권한, 로그 마스킹, 외부 API 키 관리, 사용량 통제가 필요합니다.

권장 운영 원칙: “자동 결정”이 아니라 “사람이 검토할 전략 후보 생성”으로 포지셔닝해야 합니다.
AI 의사결정 보조 도구로 쓰되, 최종 판단은 실무 검토와 실제 데이터로 보완합니다. 14

경진대회 이후 확장

실제 회사 업무에 쓰려면 내부 데이터, 피드백, 협업 흐름을 붙이는 것이 다음 단계입니다.

FP 상담 결과와 전환 여부를 기록해 추천 정확도를 실제 업무 데이터로 개선합니다.

4단계 · 운영형 데이터 계층

벡터 저장소, 권한, 감사 로그, 팀 공유 기능을 붙여 PoC에서 사내 서비스로 전환합니다.

5단계 · 규정 검토

보험 광고 심의, 금지 표현, 개인정보·AI 거버넌스 체크리스트를 자동 검토합니다.

6단계 · 실험 성과 측정

추천 타겟 캠페인과 기존 방식의 반응률을 비교해 실제 사업 효과를 계량화합니다.
