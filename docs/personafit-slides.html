Project Overview

프로젝트 개요

보험 상품의 초기 타겟 가설을 정량적으로 좁히는 의사결정 보조 플랫폼입니다.

입력은 상품설명서·약관이고, 출력은 반응 가능성이 높은 페르소나·인구통계·지역·영업 화법입니다.

대상 사용자

상품기획자, FP, 마케팅 담당자가 신규 상품 타겟과 공략 지역을 빠르게 점검합니다.

핵심 데이터

NVIDIA Nemotron-Personas-Korea 100만 합성 한국인 페르소나와 1536차원 임베딩을 사용합니다.

결과 활용

상·하위 반응 비교, 지역 분포, 설문 응답 생성, 리포트로 상품 전략 회의 자료를 구성합니다.

실제 인물 데이터가 아닌 합성 페르소나 기준 분석입니다. 03

Analysis Process

상품 분석 프로세스

문서 입력부터 리포트까지 6단계로 자동 처리합니다.

1 문서 입력 텍스트 직접 입력 또는 TXT/PDF/DOCX/HWP/HWPX 업로드
2 소구점 추출 LLM tool 호출로 타겟 조건과 혜택을 구조화
3 쿼리 임베딩 요약·혜택·키워드를 1536차원 벡터로 변환
4 전체 점수화 코사인 유사도 + 룰 보너스 + 카테고리 보너스
5 지역·계층 집계 핵심 타겟 0.5%, 타겟층 5%, 관심층 20%
6 의견·리포트 상·하위 페르소나 의견과 FP용 영업 리포트 생성
현재 스코어링 공식은 0.7 × cosine + 0.2 × rule_bonus + 0.1 × category_bonus입니다. 룰 보너스는 연령·성별·가구·학력·직업을 반영하며, 직업은 부분 매칭으로 처리합니다.
분석 결과는 JSON Lines로 저장되어 이력 화면에서 재확인됩니다. 07

How to Use

사용 방법

데모 사용자는 문서 입력, 결과 확인, 세그먼트 저장, 설문 실행 순서로 쓰면 됩니다.

STEP 1

상품 입력

홈 화면에서 약관·상품설명서를 붙여넣거나 파일로 업로드합니다. 모델은 기본 sLLM, 필요 시 Claude로 전환합니다.

STEP 2

결과 검토

요약 점수, 상위·하위 페르소나, 지역 지도, 핵심·타겟·관심층 통계, 리포트를 확인합니다.

STEP 3

대상자 확장

페르소나 탐색에서 자연어 조건과 필터로 대상을 찾고, 선택 그룹을 세그먼트로 저장합니다.

STEP 4

설문 실행

설문 마법사에서 대상·질문·모델을 설정한 뒤 진행률 화면과 응답 리포트에서 결과를 확인합니다.

화면 경로 목적
상품 분석 / 신규 분석, 분석 이력, 결과 대시보드
데이터 현황 /overview 100만 행 분포, 지역·직업·페르소나 텍스트 확인
페르소나 탐색 /personas 조건 검색, 세그먼트 저장
설문 시뮬레이션 /surveys 설문 생성, 실행, 응답, 리포트 확인
분석 이력과 설문 이력은 로컬 JSON 저장소 기준으로 유지됩니다. 10

Risks & Next Steps

리스크와 다음 단계

데모 완성도는 높지만, 실제 의사결정 도구로 쓰려면 검증 축을 추가해야 합니다.

데이터 한계 합성 페르소나가 실제 가입자 분포를 완전히 대변하지 않습니다. 외부 실제 통계와 교차 검증이 필요합니다.
전환율 한계 현재 검증은 타겟 조건 일치성입니다. 실제 가입 의향·전환율은 A/B 테스트나 사내 데이터가 필요합니다.
운영 한계 100만 행 인메모리 구조는 빠르지만 메모리 요구량이 큽니다. 운영 배포 시 벡터 DB 또는 샤딩 검토가 필요합니다.
품질 한계 LLM 응답 품질은 프롬프트·모델 상태에 좌우됩니다. 핵심 설문 유형별 회귀 테스트가 필요합니다.

다음 단계 1

실제 보험 가입자 분포와 비교하는 외부 벤치마크를 추가합니다.

다음 단계 2

Supabase 또는 벡터 저장소로 운영형 데이터 계층을 분리합니다.

다음 단계 3

설문 리포트 자동 해석, CSV 내보내기, 팀 공유 흐름을 보강합니다.

현재 산출물은 데모와 내부 보고에 적합하며, 운영 전 검증 체계 확장이 필요합니다. 14
